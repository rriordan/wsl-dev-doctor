# v0.1.0 — Initial MVP

**Status:** Draft release notes. A GitHub Release and tag have not been created.

## What shipped

- Read-only WSL/Linux developer-environment diagnostics.
- Checks for WSL detection, common developer tools, stale `PATH` entries, root disk capacity, Docker availability, TCP listeners, and NVIDIA GPU visibility.
- Safe environment-variable inventory: names only; no values are collected or emitted.
- Markdown and JSON reports with prioritized remediation.
- `--fail-on warn|fail` for shell or CI automation.
- GitHub Actions CI running formatting, linting, type checking, tests, and a real CLI invocation.

## Validation

- Local: Ruff formatting/lint, MyPy, and pytest pass.
- Hosted CI: passing on commit `f78eb89`.
- Missing optional diagnostic commands now yield an informational result instead of terminating the CLI.

## Known limitations

- Linux-side diagnostics only; no Windows-host interrogation yet.
- Docker and GPU checks require their CLIs to be available.
- The tool recommends remediation but does not apply it.

## Next candidates

- Windows companion checks via `wsl.exe --status`.
- Recorded fixtures for representative WSL and Docker scenarios.
- Opt-in, reviewable redacted support bundles.
